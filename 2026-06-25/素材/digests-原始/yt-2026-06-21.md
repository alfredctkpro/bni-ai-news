## 🟣 Agent 範式

### **「Loop Engineering 循環工程」——不再 prompt agent，而是設計會自動觸發 agent 的 loop** — @AI Jason  ⚠️🔗
🔥 3 頻道在談（AI Jason、Matthew Berman、Best Partners TV 都收了 Addy Osmani 那條）
- 🏢 **ctkpro**：這是**可直接重組工作流的範式級內容**。AI Jason 實測自己公司跑著：support loop（每 30 分鐘自動處理工單＋記下產品摩擦點到 `signals/` 資料夾）、SEO loop（每天 9am 自動發 20–40 篇高品質頁面引流）、各 loop 共用一個檔案系統（artifacts/signals/logs）形成**複利**。核心是把「prompt agent」升級成「設計 trigger＋共享記憶＋可驗證 codebase harness」。對接案/自家產品的自動化（客服、SEO、PR）有立即可抄的藍圖。
- 🧠 **me**：這是 ⑤ 能力前緣的下一關——從「會用 Claude Code」到「會設計 agent loop 系統」。
- ⚡ 可試：他開源了 `setup codebase harness` skill 與 `loop engineer setup` repo 模板（影片描述有連結，免費）。
- 🔗 [AI Jason: wtf is Loop Engineer](https://www.youtube.com/watch?v=W6x-hb44C0c)
- ⏱ **值不值得看完**：**強烈建議看完**（~20 分鐘）。資訊密度高、有具體檔案結構與實作步驟，是本期對 ctkpro 最可行動的一支。

### **「9 個 AI agent 不變趨勢」——從 prompt hack 到 super app／cloud agent／skills 自組裝** — @Riley Brown  ⚡🔗
- 🧠 **me**：對 ⑤「成為 agent native」的能力地圖最系統的一支。九趨勢：①模型變聰明→prompt hack 死、會描述就贏 ②skills 自組裝（叫 agent 把做好的事變成 skill）③super app（Codex/Claude Desktop 變超級 app）④cloud agent 進 iMessage/Slack ⑤底層軟實力（溝通/委派/品味）浮上來 ⑥非同步自動化 ⑦電腦操控 ⑧前沿貴、開源（GLM 5.2）追上 ⑨即時語音＋電腦操控。
- 🏢 **ctkpro**：super app＋cloud agent 趨勢＝客戶會問「能不能用 Slack/iMessage 跟我的 agent 對話」，是服務設計的領先指標。
- ⚡ 可試：影片有 GLM 5.2 在 Cursor 經 OpenRouter 的具體設定步驟（省 5x 成本）。
- 🔗 [Riley Brown: 9 trends](https://www.youtube.com/watch?v=vhyna9ur6Gc)
- ⏱ **值不值得看完**：**值得**（趨勢框架清楚、不過時）。但若時間緊，看趨勢 2/3/5/8 即可，其餘可快轉。

## 🔵 商業需求

### **Nadella「token capital × human capital × 學習迴圈」——AI 時代公司的新資產負債表** — @The AI Daily Brief  🏛️🔗
- 🏢 **ctkpro**（高度對口④）：Nadella 那篇被看 6500 萬次的貼文核心——**護城河不在「選哪個模型」，在「建一套會把人的判斷沉澱成可複利 IP 的學習迴圈」**。公式：token capital = human capital × scaffolding（harness/orchestration）× feedback loop，任一項為 0 則為 0。多數公司有付 Claude/GPT 但 scaffolding=0、feedback loop=0＝raw prompt into raw model，無沉澱。這直接定義了 ctkpro 對客戶的差異化價值：不是賣模型，是幫客戶建那層 applied AI harness（含私有 eval、RL 環境、機構記憶）。Accenture 本週財報暴跌 18%（十年低點）正是「只懂選 vendor、缺深度落地」被市場懲罰的活案例。
- 🏛️ **進圓桌**（board）：B 拓展往哪押＋ctkpro 護城河重新定義，值得帶進策略校準。
- 🔗 [AI Daily Brief: You don't need an AI strategy](https://www.youtube.com/watch?v=sKrr_ckIt4s)
- ⏱ **值不值得看完**：**主段值得**（Nadella 全文＋多方解讀）。前段 headlines 是 Fable 進度＋Bernie 國有化 AI 提案（噪音可略）。

### **AX／AXO：產品要同時服務「人」與「Agent」——瑞幸開放 MCP/Skill 的真正意義** — @Gary Chen  🏛️🔗
- 🏢 **ctkpro**（④需求地圖核心）：這是**接案方向的領先指標**。論點：當 AI Agent 開始替人下單，產品入口從「給人點的 App」變成「能被 agent 調用的服務」。新概念 AX（Agent Experience，Netlify CEO 提出）＝設計給 agent 用的體驗；AXO＝Agent Experience Optimization（對應 SEO，讓 agent 比較多品牌時選你）。對 ctkpro：客戶的官網/電商/Line 未來都需要「一個 AI 能調用的版本」（開 MCP/CLI/Skill、設計任務路徑、漸進授權、prompt injection 防護）——這是一條全新的接案題材。
- 👨‍👩‍👧 **family**：間接——孩子要培養的是「設計人＋agent 協作」的品味，不是只會點 App。
- 🏛️ **進圓桌**：⏳ 領先指標（中國瑞幸先行，台灣約 6–12 月後反應）→ 帶進 board 想「ctkpro 要不要提早備這條服務線」。
- 🔗 [Gary Chen: 為什麼該做產品給 AI 用](https://www.youtube.com/watch?v=41LR-NhwHfI)
- ⏱ **值不值得看完**：**值得**（華文、把趨勢拆到產品邏輯層，是少見不追新聞而講商業入口重構的內容）。
