## 🟣 Agent 範式

### **Claude Code 半年內「用法」靜悄悄三大轉變——你可能還在用半年前的笨方法** — @Simon Scrapes ⭐ ⚡🔗
🔥 併入「Claude 生態/Code 演進」熱點（9 片 / 6 頻道：Simon Scrapes、Chase AI、Futurepedia、Tina Huang、The AI Daily Brief、技术爬爬虾）

- 🏢 **ctkpro**：**這是本期對 ctkpro 工作流最直接可抄的一支**。三大轉變：①**記憶**——原生 `claw.md` 手寫太弱（memory.md 索引幾乎空），社群框架 Memarch/Gbrain/Hermes 補上 storage / 短期注入 / 語意 recall（兩行指令 plugin 裝：`/plugin marketplace add memarch` → install），長期專案/團隊才需要。②**規劃努力層**——`/effort`（low→max→**ultra code**），ultra code 會自寫 bespoke workflow、自動 spin up agent 團隊、彼此 adversarial 互查（6 種 workflow pattern 在 Anthropic docs），把「人決定怎麼拆」的活拿掉。③**自主執行**——`/goal`（設可檢查的完成條件，每回合用 haiku 小模型查是否達標）+ `/routines`（排程雲端 agent）/ `/loop`（本地 <3 天循環），人只在 checkpoint 介入。
- 🧠 **me**：⑤ 能力前緣的下一關——從「會下 prompt」到「設計 context 管理層（品牌/客戶/已做決策）+ 自主 loop」。Simon 示範用 ultra code 一句話造出「每早掃 inbox→評分→產內容點子→草擬貼文」的 skill 組，正是 thinkr 自動化（folo/yt digest 之外）可借鏡的。
- ⚡ 可試：今天就能在現有 Claude Code 上試 `/effort` 切 ultra code、用 `/goal` 包一個有明確完成條件的重複任務。
- 🔗 [Simon Scrapes: Claude Code Has Quietly Evolved](https://www.youtube.com/watch?v=4iMZA1omCkM)
- ⏱ **值不值得看完**：**值得完整看（~20 分鐘）**——三段都有立即可用的設定，是這批裡 ROI 最高的一支。輔助看 Futurepedia「Full Claude Ecosystem in One Video」做全景。

### **瓶頸已從「寫程式」移到「把 agent 的 PR merge 進 codebase」——OpenClaw 作者新工具 Crabbox** — @AI Jason ⭐ ⚠️🔗
🔥 併入「OpenClaw/Harness」訊號（AI Jason、TGLTommy 都在談 agent harness 這層）

- 🏢 **ctkpro**：**範式級洞察**——當你並行跑 10–15 個 agent（OpenClaw 作者 Peter Steinberg 的日常），它們狂產 PR，但每個 PR 都可能改 DB schema、搶 port、撞同一個本地 Supabase 實例，**驗證/合併變成新瓶頸**。Crabbox 為每個 agent work tree 開雲端隔離 sandbox（自帶 DB/dev server）、sync 你**未 commit 的 dirty diff**（不用一堆垃圾 commit）、跑 Playwright 端到端測試、自動回傳 video/screenshot 當 PR 證據。這對 ctkpro 若要走「多 agent 並行接案」是必經的基礎建設思路。
- 🧠 **me**：理解「code base harness」這層概念——給 agent 工具去**自證其工作**（evidence），是 agent 規模化的真正關卡，不是模型聰不聰明。
- ⚡ 可試：AI Jason 開源了 `crabbox setup` skill 與 `standard codebase harness` skill（可丟給 Claude Code/Codex 自動接），用 Daytona 當 sandbox provider。
- 🔗 [AI Jason: OpenClaw Creator's new secret project (Crabbox)](https://www.youtube.com/watch?v=1HkqTlXbQmQ)
- ⏱ **值不值得看完**：**團隊有並行多 agent 需求才值得看完**（~15 分鐘、偏實作）；單人開發目前用不上，知道「harness/sandbox 隔離」概念即可。

### **「LOOPS 用在所有事」＋「一句 6 字 prompt 讓 Codex 自跑 12 天 clone Excel」** — @Matthew Berman ⭐ ⚡
🔥 併入「Loop/自主執行」訊號（Matthew Berman ×2、Chase AI ×3 都在談 loop engineering）

- 🏢 **ctkpro**：兩支 Short 把「agent 長跑自主」講得很具象——①**LOOPS**：核心是「移除人類」，給一個可量測目標（例：每個頁面載入 <50ms），agent 自動逐頁優化到達標才停。②**run for days**：`/G goal clone Excel full-feature parody` 六字，Codex 開真的 Excel 逐功能 clone，跑 12+ 天到手動停（公式/排序/格式都能用）。配合 Simon Scrapes 的 `/goal`+`/loop`，這就是「設定完成條件→走開→回來收成」的工作流。
- 🧠 **me**：⑤——練「把任務寫成可被機器檢查的完成條件」是新核心技能（goal 寫得越具體可量測，自主執行越穩）。
- ⚡ 可試：把一個重複的、有清楚終點的任務（如批次優化、批次測試），用 `/goal` + 明確完成條件試跑。
- 🔗 [Matthew Berman: Use LOOPS for everything](https://www.youtube.com/watch?v=xyJtjk-WZcA)｜[AI can run for days](https://www.youtube.com/watch?v=EtxdB7gpFqY)
- ⏱ **值不值得看完**：兩支都是 Short（各 <2 分鐘），**直接看完**當概念啟發；深度設定回去看 Simon Scrapes 那支。

## 🔵 商業需求

### 🔥 熱點卡：「做產品給 AI 用」——服務從 App（給人點）→ 可被 Agent 調用（UX→AX→AXO）
🏛️ **進 board 圓桌**（接案需求地圖變動） · 🔗

- 🏢 **ctkpro**：**這是本期對 B 拓展/接案方向最重要的訊號**。Gary Chen 以瑞幸咖啡開放 MCP/CLI/Skill 為引子，講透一個趨勢：當 AI Agent 開始代替人處理日常任務，**每個服務的使用者從「人」變成「人 + Agent」**。三層框架可直接套到「我們幫客戶做什麼」：
  - **UX**（給人用）：傳統頁面/按鈕/轉化率。
  - **AX（Agent Experience）**：把 Agent 當使用者設計——功能邊界、哪步需人類授權、缺貨/付款失敗怎麼辦、prompt injection 防護。**開 API 不是終點**，要設計整條任務路徑。
  - **AXO（Agent Experience Optimization）**：類比 SEO——讓 AI 在多品牌比較時**選你的服務**把事辦完。
  - 對 ctkpro 的直球問題：**「你現在幫客戶做的網站/服務，未來是不是需要一個『AI 可調用版本』？」** 這可能是下一波接案需求（給客戶的服務加 MCP/Agent 介面）。
- 👨‍👩‍👧 **family**：⑥——小孩該培養的「與 AI 協作/設計給 AI 用」的思維，這支是很好的具象教材（AX 的概念）。
- **配套數據（Best Partners TV 解讀 Sensor Tower《State of AI 2026》）**：印證需求真實——**Agentic Shopping 已重構電商**（Amazon Rufus 用戶成交轉化率 ~40% vs 普通 20%、下單前停留 60min vs 15min；沃爾瑪 Sparky 客單價高 35%）。市場從「拉新下載」轉向「留存/付費/跨業落地」；ChatGPT 真實受眾首次跌破 50%（46%），Claude 漲到 10%（美國近 14%，靠 coding/長文），Claude 單用戶月收入 0.5→2.76 美元。
- 🔗 [Gary Chen: 為什麼你該開始做產品給 AI 用](https://www.youtube.com/watch?v=41LR-NhwHfI)｜[Best Partners: State of AI 2026 報告解讀](https://www.youtube.com/watch?v=MxpuORrjc1E)
- ⏱ **值不值得看完**：**Gary Chen 值得完整看（~12 分鐘）**——對 ctkpro 接案方向是策略級，建議帶進 board 圓桌問「要不要把『AI 可調用版本』做成一條服務線」。State of AI 2026 那支當資料庫（need 數據時回看）即可。
