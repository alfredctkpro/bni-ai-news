- **事件**：6/16 宣布，全股票、估值 $60B，預計 Q3 完成，Cursor 成 SpaceX 完全持有子公司；同日 Cursor 辦 Compile 大會攤開收購後走向。4 位 25 歲 MIT 創辦人套現約 $2.7B；run rate $4B、YoY 7x。
- 🏢 **ctkpro（高）**：Cursor 是我日常工具棧候選/同事可能在用的核心 AI coding 工具，**東家換成 SpaceX（馬斯克系，與 xAI/Grok 同陣營）**＝未來定價、模型綁定（很可能綁 Grok）、隱私/資料政策都可能轉向。要評估「Cursor 是否仍是中立可押的工具，還是會被鎖進 xAI 生態」——這直接影響「我服務客戶要不要繼續推 Cursor」。

## 🔹 Agent範式（③）

### ⚠️ 範式訊號 · 🔥 ~6 則在談 — 開源 agentic（GLM-5.2 + LangChain Deep Agents）逼近 Claude Code/Codex
- **事件**：Harrison Chase（LangChain）一週猛推 Deep Agents / `dcode`（deepagents code）＝**model-agnostic、provider-agnostic 的通用 coding agent**，搭 GLM-5.2 開源模型「remarkably good」；社群文章「Build your own Claude Code with Deep Agents」。他自評 Deep Agents 是 Codex/Claude Code 的「model agnostic、general purpose」替代方向（但提醒「別在 claude code/codex 裡跑 GLM，那些 harness 過度調校過」）。
- 🏢 **ctkpro（中高）**：**「自建可換底層模型的 coding agent」門檻正在崩**——若開源模型（GLM-5.2）+ 開源框架（Deep Agents）能逼近 Claude Code，代表 ① 不必被單一閉源工具綁死 ② 客戶若要求「本地/自主可控的 coding agent」變得可行（資料不外流的接案賣點）。值得實測一輪：GLM-5.2 + dcode 對比現用 Claude Code 的落差有多大。
- 🧠 **me**：個人能力前緣的關鍵問句從「哪個工具最強」轉成「我能不能組裝出 model-agnostic 的 agent 工作流」——這是抗收編的真 edge（呼應上面 Cursor 卡）。
- 多方觀點：@hwchase17「dcode general purpose、model agnostic」｜@Sydney Runkle「open models 又一個 moment，GLM-5.2 配 deepagents 很猛」｜社群「build your own Claude Code」。
- 🔗 [hwchase17 GLM-5.2 實測串](https://x.com/hwchase17)（Harrison Chase 6/19–6/22 連續多則）

### ⚠️ 範式訊號 — Sakana Fugu：把「多代理協作」打包成單一 API
- **事件**：日本 Sakana AI 發表 Sakana Fugu，將 multi-agent orchestration（自選模型、分派任務、驗證）整套封進**一個模型/一個 API**，使用者像呼叫單一模型那樣呼叫，背後是一整套協作系統。
