## 🟢 工具用法

### 熱點卡 1 ·「Skills 管理工法」一次補齊（🔥 宝玉 4 連發乾貨）

- **把「Skill 該裝哪、怎麼更新、怎麼版控」講成一套可抄的工序**
  - 🏢 ctkpro：這是把 Claude Code/Codex 的 Skill 用法收斂成**團隊接案的 agent 工序規範**的現成藍本——能直接寫進你給 ctkpro 開發流的 SOP（哪些 Skill 全局/項目、怎麼避免吃爆 context、怎麼跟客戶專案 git 同步）；🧠 me：把「Skill 維運」當成像管 dotfiles 一樣的肌肉
  - 多方做法（同一人多角度，可挑一套照抄）：
    - **只裝項目內、不裝全局**：最主要為**節約上下文空間**，全局 Skill 會無謂佔 context ｜ [原文](https://x.com/dotey/status/2069632132431929651)
    - **軟連結（symlink）指向同一位置**：方便邊用邊迭代，任一專案發現問題就改原始位置、全專案同步更新；別人的 Skill 則 fork 一份再定期同步 upstream ｜ [原文](https://x.com/dotey/status/2069825516459139579)
    - **跟著 git 走**：開源/私有 repo 管理，要更新 `git pull`、要特定版 `git checkout`；甚至在 Codex/CC 裡掛**定時任務自動更新** ｜ [原文](https://x.com/dotey/status/2069675621332972027)
    - **配套兩個現成可用 Skill**（《图解Skill》repo）：`interview-analysis`（播客/影片訪談→結構化寫作素材包＋提綱、抽金句）＋另一個 illustrate 類 ｜ [原文](https://x.com/dotey/status/2069918721972523191)
  - ⚡ 可立刻試：選「項目內＋symlink＋git」這套，建一個你自己的 skills repo 管理 ctkpro 常用 Skill

### 熱點卡 2 · Claude Tag：Claude 變成常駐 Slack 的「同事」（⚠️ 翻盤訊號 · 🔥 3 人在談 · **標 Agent範式櫃接手**）

- **Anthropic 發 Claude Tag——在 Slack 頻道 @Claude 就能派任務，它自己拆步驟、逐步完成、在 thread 交付**
  - 🏢 ctkpro：這不只是個工具，是「人＋agent 協作介面」從「開 IDE 用工具」位移到「在團隊溝通管道裡指揮一個同事」——若成形，會改你（及客戶團隊）導入 AI 的切入點，從「教大家用 CC」變「把 agent 加進 Slack 工作流」；🧠 me：練「管理 agent 同事」而非「操作 agent 工具」的心態
  - 多方觀點：
    - 宝玉：以 research preview 提供，@Claude 派任務→自動拆步→thread 交付 ｜ [原文](https://x.com/dotey/status/2069477417278730536)

- **mymind 出 MCP／API：素材收集庫直接接 agent，找參考不用人工搬運** — 歸藏實測，自動化了內容創作的素材檢索環節。🏢 ctkpro：若你/團隊用素材庫做設計或內容接案，MCP 化＝agent 能自己撈參考；🧠 me：留意「個人知識/素材庫 → MCP → agent 自取」這個正在普及的接法 ｜ 歸藏 @op7418 ｜ [原文](https://x.com/op7418/status/2069615413517062280) ⚡
- **transitions.dev：可裝成 Skill，讓 agent 自動幫網頁加炫酷過渡動畫**（`npx skills add jakubantalik/transitions.dev`） — 🏢 ctkpro：做官網/landing page 接案時，agent 一鍵套高質感動畫＝省前端微調工時；🧠 me：收進「網頁交付」的 Skill 清單 ｜ 宝玉 @dotey ｜ [原文](https://x.com/dotey/status/2069513317693964418) ⚡
